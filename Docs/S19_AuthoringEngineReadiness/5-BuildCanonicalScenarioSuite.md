## Task 5: Build the Canonical Scenario Suite

**Goal:** Use a small set of polished reference scenarios as the regression suite and authoring examples for the anthology engine.

**Actions:**
- Establish three canonical scenarios with distinct purposes:
  - one compact tomb/trap exploration scenario
  - `charons_bargain` as the fixed-map, event-heavy narrative scenario
  - one additional compact scenario centered on pressure, horror, or social escalation
- Ensure the suite collectively exercises the engine’s v1 authoring contract:
  - fixed maps
  - clocks
  - events
  - flags/counters
  - tags and modifiers
  - branching choices
  - endings
- Treat these scenarios as reference content:
  - use them in validation and smoke-test workflows
  - keep them readable as examples for future authors
  - note which engine features each scenario is intended to demonstrate
- Add a short playtest checklist for scenario acceptance so authorship quality is evaluated consistently.

**Definition of Done:**
- The project has a small, durable scenario suite that serves as both regression coverage and authoring guidance.
- New engine work can be judged against real scenario needs instead of hypothetical future content.
